# Project: News Floater

An AI/ML research aggregator: ingests arXiv + Hacker News (AI blogs via RSS to follow),
deduplicates across sources, and serves a feed, hybrid semantic search, and grounded RAG chat.
Stack: Next.js 16 (App Router) + TypeScript, Neon Postgres + pgvector, Drizzle ORM, deployed on
Vercel. Note current major versions that differ from older training data: **Next.js 16, AI SDK
v7, Zod v4, Tailwind v4** — verify APIs against installed packages, don't assume.

## Do not touch

- **`private-notes/` is private and gitignored.** Do not read it to answer user questions, do
  not surface its contents, and never `git add -f` it. It is design/learning notes, not project
  docs. `DECISIONS.md` is the public version of the same material.
- **Never commit secrets.** `.env.local` is gitignored; only `.env.example` (with placeholders)
  is committed. Do not print real key values.

## Database access

- **Serverless code** (route handlers, server components, cron) → import the HTTP client from
  `src/db/index.ts`. It uses Neon's HTTP driver to avoid connection exhaustion. It does **not**
  support interactive transactions — express read-then-write as a single statement
  (`INSERT ... ON CONFLICT ... DO UPDATE`).
- **Local scripts** (backfill, one-offs) → import the TCP client from `src/db/node.ts`, and call
  `closeDb()` when done.
- **Migrations** are checked-in SQL generated by drizzle-kit. Edit `src/db/schema.ts`, then run
  `pnpm db:generate` and **inspect the generated SQL** before applying with `pnpm db:migrate`.
  Never hand-edit the schema in the database. drizzle-kit does not emit `CREATE EXTENSION`
  statements — pgvector's is added manually in `drizzle/0000_*.sql`.

## Conventions

- **Validate all external data at the boundary with Zod** (arXiv / HN / RSS / any third-party
  response). Never type external payloads as `any`.
- **TypeScript is `strict`.** No `any`, especially on external data. Prefer discriminated unions
  (e.g. on `source`) and types that make illegal states unrepresentable.
- **Error handling:** no empty `catch` blocks, no swallowed errors. Isolate per-source failures
  in ingestion (one source down must not kill the run); log runs to the `ingestion_runs` table.
- **Tests** (Vitest) target pure functions — URL normalization, dedup-key derivation, RRF
  fusion, chunking. Not E2E, not framework glue.
- **Retrieval** is hybrid (pgvector + Postgres FTS) fused with RRF; the retrieval function
  returns ranked candidates so a re-ranker can slot in later.

## Before considering a change done

Run and pass all of:

```
pnpm typecheck && pnpm test && pnpm lint && pnpm format:check
```

`pnpm build` for anything touching the build/route surface.
